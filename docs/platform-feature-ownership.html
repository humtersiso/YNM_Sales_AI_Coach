銷售顧問智慧訓練系統 — 功能與責任分工

技術驗證已完成；目前為 MVP 試用版（部署給測試人員、資料寫入 BQ、依回饋迭代）。 正式環境分工：總管理處負責 Agent 與 API，協力廠商負責業代前台與對外串接，裕日提供內容與營運決策。

三個 Agent 在做什麼（上線目標）

銷售助手 Agent

協力廠商前台把業代問題送進來 → 查 BigQuery 題庫 → 有標準話術就回覆；沒有就告訴前台「題庫沒有」，由廠商去跑專家補題流程。

MVP：網頁銷售助手可試用

對練助手 Agent

晨會或訓練時模擬客戶。內含兩個子 Agent：

  • 出題／互動 Agent：出情境、跟業代對話
  • 評分 Agent：依整段對話給分與建議

MVP：待裕日提供素材（5 月起蒐集）

資料處理 Agent

總管理處後端「資料整理伺服器」：題庫維護、新題比對、待釐清與法務、專家都回完後用 LLM 統整成建議話術等。

MVP：部分完成，6 月中可試用

# 區塊 功能 白話說明 MVP 分工 備註
A. 前台（MVP 為本專案響應式網頁；正式上線畫面由協力廠商）
A1前台銷售助手 業代發問，AI 依題庫回覆話術重點；題庫沒有則引導「待新增題庫」 已有 協力廠商 MVP 網頁已可發問；串總管理處 C1 API。正式版體驗由廠商優化
A2前台對練助手 AI 當客戶出題，業代回答；結束後看評分與建議 部分 協力廠商 MVP 有對練畫面；後端 D Agent 待建，尚無完整對練流程
A3前台使用入口 手機／電腦響應式網頁（MVP）；正式版管道待與裕日、廠商確認 已有 協力廠商 MVP 即本專案網頁入口
B. 裕日總部管理介面（智慧行銷部等 — 架構圖「管理介面」，待細部訪談）
B1裕日總部使用統計 各據點、業代使用銷售／對練的次數與分布 部分 協作 圖表多由協力廠商做；數字須來自 BQ。上線前要把詢問紀錄、對練紀錄寫入 BQ（總管理處建表＋收 log API，前台由廠商送）
B2裕日總部戰力儀表板 訓練情形與業績連動、據點排行 部分 協力廠商 畫面廠商；業績串裕日 API；訓練數據來 BQ
B3裕日總部權限與資料維護 誰能進後台、基本資料維護 部分 協力廠商 串裕日權限 API
B4裕日總部競品熱門問題 競品相關提問排行（如 Top50） 部分 協作 總管理處 BQ 彙總；廠商做圖表
B5裕日總部題庫總覽 智慧行銷部檢視 BigQuery 話術表 已有 總管理處 MVP 已改為只讀 BQ
B6裕日總部匯入與比對 上傳 Excel、檢查是否重複、哪些要進待釐清 部分 協作 比對邏輯總管理處；畫面可由廠商重做
B7裕日總部新題流程追蹤 看待釐清進度；專家回覆收齊後，總管理處統整成建議話術，交智慧行銷部審核 部分 協作 專家通知／回信由協力廠商；統整 API 總管理處；審核決策在裕日總部
C. 銷售助手 Agent（總管理處 — 給協力廠商 API 呼叫）
C1銷售 Agent題庫查詢與回覆 收到業代問題 → 查 BQ → 有則回標準話術與引用；沒有則回「題庫無」等旗標 已有 總管理處 協力廠商不直接查 BQ
D. 對練助手 Agent（總管理處 — 含子 Agent）
D0對練 Agent整體編排 協調出題互動與評分，對練紀錄寫入 BQ 規劃中 總管理處 待裕日提供對練素材；起迄看資料品質
D1子 Agent出題／互動 Agent 依情境出題、扮演客戶與業代多輪對話 規劃中 總管理處 前台由廠商呈現對話 UI
D2子 Agent評分 Agent 依業代與 AI 的整段互動結果評分、給改善建議 規劃中 總管理處 與出題 Agent 分開，方便調 prompt／規則
E. 資料處理 Agent（總管理處 — 後端資料整理伺服器）
E1資料處理題庫無 — 協作約定 銷售 API 回傳「題庫沒有」時，由協力廠商去通知專家、開待釐清流程 已有 總管理處 API 契約要寫清楚；通知執行在廠商
E2資料處理專家回覆後 LLM 統整 多位專家都回完後，廠商呼叫統整 API，產出建議話術 已有 總管理處 MSD 整合 API
E3資料處理新題匯入 BQ 話術 Excel 匯入、寫入 staging／正式表 已有 總管理處 裕日內容＋管線
E4資料處理重複比對 新問題與題庫比對，決定是否重複、是否進待釐清 已有 總管理處 以 BQ 題庫為準
E5資料處理法務審核（流程） 法務透過連結填寫核准／駁回；決策狀態由總管理處 API 記錄 已有 協作 寄信、連結由協力廠商；法務為裕日單位
E6資料處理法務通過 — 寫回 BQ 題庫 核准且智慧行銷部確認後，將標準話術寫入 BQ 正式話術表 部分 總管理處 MVP 決策先寫試用 store；上線由總管理處提供寫回 BQ API／管線
E7資料處理專家分派規則 哪一題要給哪位專家（與菁英團隊營運規則銜接） 部分 協作 名單與營運裕日；系統規則總管理處
F. 資料層（BigQuery 知識庫）
F1資料話術主表 標準話術、客戶疑問、審核欄位等 已有 總管理處 裕日 DBA／法遵共管
F2資料其他訓練資料 FAQ、品牌／車型／競品等 部分 裕日 內容裕日；入庫總管理處
F3資料詢問／對練紀錄 業代問了什麼、對練場次與得分等，供 B1／B4 統計 部分 總管理處 上線必建：總管理處 BQ 表＋寫入 API；協力廠商前台／Agent 呼叫寫入
G. 對外系統（協力廠商串接：裕日 API、寄信等）
G1對外裕日登入／權限 業代與管理者身份、據點 部分 協力廠商 規格待裕日
G2對外裕日業績 戰力儀表板用 規劃中 協力廠商 —
G3對外寄信／通知專家 待釐清、法務信、專家回信等 部分 協力廠商 SendGrid 等；信內可附總管理處 token 連結

sequenceDiagram
  participant Agent as 業代
  participant Vendor as 協力廠商前台
  participant HQ as 總管理處API
  participant BQ as BigQuery
  participant Expert as 專家流程
  participant Mkt as 智慧行銷部

  Vendor->>HQ: 銷售問答 API
  HQ->>BQ: 查題庫
  alt 題庫有
    HQ-->>Vendor: 回話術
    Vendor-->>Agent: 顯示回答
    Vendor->>HQ: 寫入詢問紀錄（F3）
    HQ->>BQ: log
  else 題庫無
    HQ-->>Vendor: 題庫無加旗標

    Expert-->>Vendor: 專家都回完
    Vendor->>HQ: LLM 統整 API
    HQ-->>Vendor: 建議話術
    Vendor->>Mkt: 智慧行銷部審核
    Mkt->>HQ: 法務通過後寫回題庫（E6）
